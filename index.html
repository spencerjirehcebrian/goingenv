Overview

GoingEnv is a command-line tool that helps you securely manage environment files across different projects and environments. It scans, encrypts, and archives your .env files with military-grade AES-256 encryption.

Quick Start

Your First Backup

Navigate to Your Project

cd /path/to/your/project

Check What Files Are Detected

goingenv status

This shows you all the environment files GoingEnv can find in your project.

Create Your First Backup

goingenv pack -k "your-secure-password"

This creates an encrypted archive of all your environment files.

Verify Your Backup

goingenv list -f backup.enc -k "your-password"

This shows the contents of your encrypted archive without extracting files.

Interactive Mode (Recommended for Beginners)

For a user-friendly experience, simply run:

goingenv

This launches the interactive terminal interface where you can navigate using arrow keys and follow the guided workflow.

Step-by-Step Tutorials

Tutorial 1: Setting Up Environment Backup

Learn how to create your first environment backup and restore it.

Prepare Your Project

Create a sample project with environment files:

mkdir my-project && cd my-project
echo "DATABASE_URL=postgres://localhost:5432/myapp" > .env
echo "API_KEY=demo-key-12345" > .env.local
echo "NODE_ENV=development" > .env.development

Scan for Environment Files

Check what GoingEnv detects in your project:

goingenv status

You should see output showing the 3 files we just created.

Create Encrypted Backup

Create your first encrypted backup:

goingenv pack -k "my-secure-password" -o my-project-backup.enc

Verify Backup Contents

List the contents of your backup without extracting:

goingenv list -f my-project-backup.enc -k "my-secure-password"

Test Restoration

Move to a new directory and restore your files:

mkdir ../restore-test && cd ../restore-test
goingenv unpack -f ../my-project/my-project-backup.enc -k "my-secure-password"
ls -la

Tutorial 2: Team Environment Sharing

Learn how to share environment configurations securely with your team.

Create Team Template

As a team lead, create a standardized environment template:

Share Template

Share the team-env-template.enc file and password with team members through secure channels.

Team Member Setup

New team members can restore the environment:

goingenv unpack -f team-env-template.enc -k "team-shared-password"

Customize Local Environment

Create personal environment overrides:

cp .env .env.local
# Edit .env.local with personal API keys, local database URLs, etc.

Tutorial 3: Deployment Environment Management

Manage different environments for development, staging, and production.

Create Environment-Specific Backups

# Development environment
goingenv pack -k "dev-password" -o dev-env.enc

# Staging environment
goingenv pack -k "staging-password" -o staging-env.enc

# Production environment
goingenv pack -k "prod-password" -o prod-env.enc

Deploy to Different Environments

Switch between environments by restoring different backups:

# Deploy to staging
goingenv unpack -f staging-env.enc -k "staging-password" --force

# Deploy to production
goingenv unpack -f prod-env.enc -k "prod-password" --force

Automated Deployment Script

Create a deployment script:

#!/bin/bash
ENVIRONMENT=${1:-development}
PASSWORD_FILE="~/.goingenv/${ENVIRONMENT}-password"

if [[ -f "$PASSWORD_FILE" ]]; then
    PASSWORD=$(cat "$PASSWORD_FILE")
    goingenv unpack -f "${ENVIRONMENT}-env.enc" -k "$PASSWORD" --force
    echo "Deployed $ENVIRONMENT environment"
else
    echo "Password file not found: $PASSWORD_FILE"
    exit 1
fi

Core Commands

Status Command

Pack Command

Create encrypted archives of environment files.

Basic Pack

Specific Directory
goingenv pack -k "password" -d /path/to/project

Unpack Command

Restore files from encrypted archives.

Basic Unpack
goingenv unpack -f backup.enc
goingenv unpack -f backup.enc -k "password"
Custom Target
goingenv unpack -f backup.enc -k "password" -d /restore/path

List Command

View contents of encrypted archives without extracting.

List Archive
goingenv list -f backup.enc -k "password"
Detailed Listing
goingenv list -f backup.enc -k "password" --verbose

Real-World Examples

Daily Backup Script

Automate daily backups of all your projects:

#!/bin/bash
# daily-env-backup.sh

PROJECTS_DIR="$HOME/projects"
BACKUP_DIR="$HOME/backups/env-files"
DATE=$(date +%Y%m%d)

mkdir -p "$BACKUP_DIR"

for project in "$PROJECTS_DIR"/*; do
    if [[ -d "$project" ]]; then
        project_name=$(basename "$project")
        backup_file="$BACKUP_DIR/${project_name}_${DATE}.enc"
        
        echo "Backing up: $project_name"
        cd "$project"
        
        if goingenv pack -k "$BACKUP_PASSWORD" -o "$backup_file"; then
            echo "‚úÖ $project_name backed up successfully"
        else
            echo "‚ùå Failed to backup $project_name"
        fi
    fi
done

CI/CD Integration

GitHub Actions workflow for environment backup:

name: Environment Backup
on:
  push:
    branches: [main]

jobs:
  backup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      
      - name: Install GoingEnv
        run: |
          curl -sSL https://raw.githubusercontent.com/spencerjirehcebrian/goingenv/main/install.sh | bash -s -- --yes
      
      - name: Create Environment Backup
        run: |
          goingenv pack -k "${{ secrets.BACKUP_PASSWORD }}" -o "production-backup.enc"
        env:
          BACKUP_PASSWORD: ${{ secrets.BACKUP_PASSWORD }}
      
      - name: Upload Backup Artifact
        uses: actions/upload-artifact@v3
        with:
          name: environment-backup
          path: ~/.goingenv/production-backup.enc

Environment Migration

Migrate environment configurations between servers:

# On source server
cd /app
goingenv pack -k "migration-key" -o migration.enc

# Transfer migration.enc to target server

# On target server
cd /app
goingenv unpack -f migration.enc -k "migration-key"
goingenv status  # Verify restored files

Multi-Environment Workflow

Manage multiple environment configurations:

#!/bin/bash
# deploy.sh

ENVIRONMENT=${1:-development}

case $ENVIRONMENT in
  development)
    goingenv unpack -f dev-env.enc -k "$DEV_PASSWORD"
    ;;
  staging)
    goingenv unpack -f staging-env.enc -k "$STAGING_PASSWORD"
    ;;
  production)
    goingenv unpack -f prod-env.enc -k "$PROD_PASSWORD"
    ;;
  *)
    echo "Unknown environment: $ENVIRONMENT"
    exit 1
    ;;
esac

echo "Deployed $ENVIRONMENT environment"
